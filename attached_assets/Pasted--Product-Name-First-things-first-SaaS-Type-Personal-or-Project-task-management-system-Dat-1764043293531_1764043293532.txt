**Product Name:** First things first
**SaaS Type:** Personal or Project task management system
**Database:** PostgreSQL
**GUI Style:** Bloomberg Terminal / IBM 3270 Mainframe Terminal 

# Pages
1. Displays only the highest priority task from the ordered list
2. Ordered list of all tasks associated with the milestones.
3. Milestones. Max of 5. Presented as columns with tasks underneath.

# Data relationships:
Users have many child milestones.
Milestones have many child tasks.
Tasks have one parent Milestone.

## Authentication
**Type:** Replit Auth
**User Constraints:**
- Users can not see other users' milestones or tasks.
- Each user will have a unique milestones and tasks table in the database.

# System Design

**Design condition:** Each user will have a unique milestones and tasks table in the database.
## Database Schema

### **users**
- `id` (UUID, Primary Key)
- `email` (VARCHAR(255), Unique, Not Null)
- `password_hash` (VARCHAR(255), Not Null)
- `created_at` (TIMESTAMP, Default: NOW())
- `updated_at` (TIMESTAMP, Default: NOW())

### **milestones**
- `id` (UUID, Primary Key)
- `user_id` (UUID, Foreign Key → users.id)
- `title` (VARCHAR(255), Not Null)
- `description` (TEXT)
- `definition_of_done` (TEXT)
- `display_order` (INTEGER, Not Null)
- `is_deleted` (BOOLEAN, Default: false)
- `deleted_at` (TIMESTAMP, Nullable)
- `created_at` (TIMESTAMP, Default: NOW())
- `updated_at` (TIMESTAMP, Default: NOW())

### **tasks**
- `id` (UUID, Primary Key)
- `milestone_id` (UUID, Foreign Key → milestones.id)
- `title` (VARCHAR(255), Not Null)
- `description` (TEXT)
- `definition_of_done` (TEXT)
- `milestone_order` (INTEGER, Not Null) *(order within milestone, max 10)*
- `global_order` (INTEGER, Not Null) *(order in the global task list)*
- `is_completed` (BOOLEAN, Default: false)
- `completed_at` (TIMESTAMP, Nullable)
- `is_deleted` (BOOLEAN, Default: false)
- `deleted_at` (TIMESTAMP, Nullable)
- `created_at` (TIMESTAMP, Default: NOW())
- `updated_at` (TIMESTAMP, Default: NOW())

### **trash**
- `id` (UUID, Primary Key)
- `user_id` (UUID, Foreign Key → users.id)
- `item_type` (VARCHAR(50), Not Null) *(milestone or task)*
- `item_id` (UUID, Not Null)
- `item_data` (JSONB, Not Null) *(snapshot of deleted record)*
- `deleted_at` (TIMESTAMP, Default: NOW())
- `expires_at` (TIMESTAMP, Not Null) *(deleted_at + 30 days)*
### Database Key Design Notes:

1. **Dual ordering for tasks**: `milestone_order` (position within milestone) and `global_order` (position in global prioritized list)
2. **Soft deletes**: `is_deleted` and `deleted_at` flags keep records for 30-day trash period
3. **Trash table**: Stores JSONB snapshot of deleted items for recovery
4. **Max 5 milestones**: Enforced at application level via constraint on `display_order` (1-5)
5. **Max 10 tasks per milestone**: Enforced at application level via constraint on `milestone_order` (1-10)
6. **UUIDs**: Used for primary keys for better distributed system support

## UI/UX (Terminal-Inspired Design)
- Monochrome style with high contrast (green/white on black background). 
- Keyboard-driven navigation. 
- Modal popups for editing records (IBM 3270-inspired). 
# Features:
1. Modal opens when a task or milestone is selected
2. Option to delete in the modal and click Complete.
3. Milestone modal has displays fields in two columns. The first column has a field for "Description" and "Definition of Done". The second column has a list of a maximum of 10 tasks, which can be ordered by drag and drop. 
4. Task modal has two fields: "Description" and "Definition of Done"
5. Deleted milestones or tasks are stored in the trash for 30 days
6. Tasks on the ordered list page can be rearranged by dragging and dropping tasks. Their priority is unaffected by the priority order selected within a milestone.
